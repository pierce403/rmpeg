# 2026-06-14 00:10:05 UTC Matroska/WebM Metadata

Objective: improve Phase 1 strict media matches with conservative Matroska/WebM track metadata probing.

Change:

- Added an EBML/Matroska parser that requires `DocType` `matroska` or `webm`.
- Parses `Segment` -> `Tracks` -> audio/video `TrackEntry` metadata.
- Maps common Matroska/WebM codec IDs for VP8, VP9, AV1, H.264, HEVC, Opus, Vorbis, AAC, FLAC, MP3, and little-endian PCM.
- Requires a `Cluster` element before accepting a file to avoid accepting WebM DASH header fragments as standalone media.

Observed corpus impact:

- Before: 754 / 2178 strict media matches = 34.6%.
- After: 1006 / 2178 strict media matches = 46.2%.
- Gain: +252 strict media matches.
- Total corpus passes: 1338 / 2511.
- Corpus errors: 0.
- False accepts: unchanged at 1 known AAC/USAC case.
