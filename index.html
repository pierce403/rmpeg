rmpeg

rmpeg is an exploratory Rust reimplementation of FFmpeg, guided by autoresearch methodology: study the existing system, decompose its behavior, and rebuild the core ideas with a smaller, safer, and more inspectable architecture.

  • Rust-first media tooling
  • FFmpeg-compatible ambition
  • Research-driven implementation

Project Direction

FFmpeg is broad, battle-tested infrastructure. rmpeg starts from the opposite end: targeted experiments that map codecs, containers, filtering, timing, and command behavior into Rust modules with explicit boundaries and testable contracts.

The goal is not a quick clone. The goal is to use automated research, trace analysis, differential testing, and focused Rust implementations to learn which parts of FFmpeg can be rebuilt clearly without losing the practical behavior users rely on.

Research First

Capture observed FFmpeg behavior, public format specifications, and compatibility edge cases before committing to APIs.

Rust Core

Prefer memory-safe parsing, explicit error handling, and modular crates over large shared global state.

Compatibility Loop

Compare rmpeg output against FFmpeg wherever possible, using tests and fixtures to keep behavior grounded.
